Introduction

This is a multipurpose module for Kemana Directory. This module contains support for featured items, most visited, best rated, and more.

[ back to top ]

Installing

To install this Module in, follow these easy steps:

  1. Go to Admin CP > Configure Site > Modules.
  2. Click on Scan New Module.
  3. On 'Available Modules', click 'Documentation' to read this manual.
  4. Click on 'Install' to begin install process.

You may also need to change your template, see Usage section.

Usage

This module is a multipurpose module for Kemana (KeCM). There are various features in this module to enrich the Kemana experience.

You can use Module Manager or edit the template directly to add this module to your Kemana installation.

<!-- BEGINMODULE ke_core -->
mode = item_list
dir_id = dir_id
items = ( random | best | newest | see_also | cat_featured | dir_featured | site_featured | history | list_of_item_id )
limit = number of products to display
random = ( true | false )
user_id = user_id
item_status = ( P | E | T )
item_visibility = ( A | M | H )
display = ( grid | list | list_small )
csswrapper = css_class
div_id = div_id
csswrapper_grid = css_class
csswrapper_list = css_class
<!-- ENDMODULE ce_core -->

Parameters:

  • mode: item_list only for now.
  • dir_id: directory short name, eg: gen, hom, aut, etc (optional).
  • items
    • random: display random items
    • best: display most visited items
    • newest: display newest items
    • see_also: display 'see also' items for a specific item (in detail screen)
    • cat_featured: display featured items for a specific category (in list screen)
    • dir_featured: display featured items for a specific directory (in welcome screen)
    • history: display history of user visited items
    • user_id: display items belong to a specific user_id, define the user_id in user_id parameter (see below)
    • list_of_item_id, eg: 1,2,3,4,5: display specific items, separated with commas.
  • limit: 10 = limit number of products to display (default 10).
  • random: set 'true' to display random items instead if 'item' in items not available (default false).
  • user_id: filter results by user_id, to be used with parameter: items = user_id
  • item_status: filter results by status, which are:
    • P: all published items (default)
    • E: all pending items
    • T: all items still require user confirmation
    • ALL: all items still require user confirmation
  • item_visibility: filter results by item visibility, which are:
    • A: available to all members & guests (default)
    • M: available to members only
    • H: hidden items
    • ALL: all
  • display
    • grid: display in grid style (default)
    • list: display in list style
    • list_small: display in vertical list with small thumbnail
  • csswrapper: your own custom css for output wrapping (optional).
  • div_id: name of the container id (optional). If set, ke_core module will display a shortcut to change display mode (grid or list) which is clickable by visitors.
  • csswrapper_grid & csswrapper_list: only required if you set div_id. CSS class for respective display mode.

Examples:

Displaying most visited items from General Directory:

<!-- BEGINMODULE ke_core -->
mode = item_list
dir_id = gen
items = best
display = grid
<!-- ENDMODULE -->

Displaying specific items from all directory:

<!-- BEGINMODULE ce_core -->
mode = item_list
items = 1,2,3,4,5
display = list
<!-- ENDMODULE -->

Displaying pending items from all directory for a specific user_id:

<!-- BEGINMODULE ce_core -->
mode = item_list
items = user_id
user_id = admin
item_status = E
item_visibility = ALL
display = list
<!-- ENDMODULE -->

Displaying random items with visitors clickable display mode:

<div id="mydiv"> <!-- BEGINMODULE ce_core -->
mode = item_list
items = random
display = list
div_id = mydiv
csswrapper_grid = col-sm-4 col-md-3
csswrapper_list = col-sm-12
<!-- ENDMODULE -->
</div>

Uninstalling

To uninstall this Module in, follow these easy steps:

  1. Go to Admin CP > Configure Site > Modules.
  2. Find Page Gallery. Click Uninstall.
  3. You can choose to also remove database & files.

Uninstalling may require you to manually remove some files for 100% clean uninstall.
